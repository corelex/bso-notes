2. BSO Ontology - Events

BSO has about 1200 event types. It seems that the event hierachy is in worse shape than the entity hierarchy. Many design decisions were constrained by the intended application. We may not be able to salvage a lot.

The new event hierarchy will start off as a very flat structure, depth can be added when CPA informs us that several entries can be grouped. That is, structure of the hierarchy is governed not by theoretical constraints but by distribution data in the corpus.

Addition of entries is driven by CPA:

  1. We'll start with the approx. 1000 patterns that CPA has as of now, checking what BSO event types they refer to and creating a small hierarchy from them. Some of this work has already been done by Patrick and Anna in Berlin.

    Should we immediately add the 100 verbs that carry the patterns?

  2. Then take the PropBank verbs and NomBank relations and choose types from them, renaming and creating types at will.
  3. Use analyses from CPA to determine role structure.
